
# TRADEPO - Инвестиционная Платформа

## Описание проекта

TRADEPO - это современная инвестиционная платформа, предоставляющая возможность пользователям инвестировать средства в различные тарифные планы и получать ежедневную прибыль. Проект включает многоязычный интерфейс (английский, русский, казахский и другие языки), админ-панель для управления пользователями, депозитами и выводами средств, а также реферальную систему.

### Основные функции

- **Многоязычный интерфейс**: поддержка нескольких языков (EN, RU, KZ, UZ, TJ)
- **Расчет прибыли**: калькулятор для расчета потенциальной прибыли от инвестиций
- **Тарифные планы**: различные инвестиционные тарифы (Базовый, Премиум, Элит)
- **Личный кабинет пользователя**: управление депозитами, выводами и рефералами
- **Админ-панель**: управление пользователями, транзакциями и контентом сайта
- **Реферальная система**: возможность привлечения новых пользователей и получения бонусов

## Технологический стек

- **Frontend**: React, TypeScript, Tailwind CSS, Radix UI
- **Backend**: Node.js, Express
- **База данных**: Drizzle ORM, Neon PostgreSQL
- **Аутентификация**: JWT, Passport.js
- **Локализация**: i18next

## Установка и запуск

### Требования

- Node.js (версия 16+)
- npm или yarn

### Установка зависимостей

```bash
# Установка зависимостей проекта
npm install
```

### Запуск проекта в режиме разработки

```bash
# Запуск клиентской части (Frontend)
npm run dev
```

Или используйте существующие настроенные воркфлоу в Replit:

1. Нажмите кнопку "Run" для запуска Dev Server
2. Приложение будет доступно по URL вашего Replit проекта

### Сборка проекта для production

```bash
# Сборка проекта
npm run build

# Запуск в production режиме
npm run start
```

## Локализация

Проект поддерживает несколько языков:

- Английский (EN)
- Русский (RU)
- Казахский (KZ)
- Узбекский (UZ)
- Таджикский (TJ)

Файлы локализации находятся в директории `client/src/i18n/locales/`.

## Возможные проблемы и решения

При запуске проекта могут возникнуть следующие ошибки:

1. **Ошибки в файлах локализации** - проверьте синтаксис в файлах локализации, особенно в `ru.ts` и `kz.ts`, где могут быть дубликаты ключей.
2. **Порт уже используется** - убедитесь, что порт 5000 не занят другими приложениями.
